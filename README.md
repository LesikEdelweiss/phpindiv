# Simple Blog (PHP MVC-lite)

## Описание проекта

Данное веб-приложение представляет собой простой блог, разработанный с использованием PHP, MySQL и Docker.  
Проект реализует базовые функции социальной платформы: регистрацию пользователей, авторизацию, создание постов и комментариев, а также административную панель.

---

## Инструкции по запуску проекта

### 1. Клонирование репозитория
```bash
git clone https://github.com/LesikEdelweiss/phpindiv
cd phpindiv
````

---

### 2. Запуск через Docker

```bash
docker compose up --build
```

---

### 3. Доступ к приложению

* Сайт: [http://localhost:8080](http://localhost:8080)
* phpMyAdmin: [http://localhost:8081](http://localhost:8081)

---

### 4. База данных

Импортируйте файлы:

```
sql/init.sql
sql/create_admin.sql
```

или через phpMyAdmin.

---

## Цель работы

Разработать веб-приложение средней сложности с серверной логикой, включающее:

* аутентификацию пользователей
* работу с базой данных
* формы ввода данных
* разделение прав доступа
* административную панель

## Функциональные возможности

### Пользователи

* регистрация
* авторизация
* выход из системы

### Посты

* создание постов
* просмотр всех постов
* просмотр отдельного поста
* поиск постов

* добавление комментариев к постам

* создание администраторов

## Сценарии взаимодействия пользователей

### 1. Обычный пользователь

1. Регистрируется
2. Входит в систему
3. Просматривает посты
4. Оставляет комментарии

---

### 2. Администратор

1. Входит в систему с ролью admin
2. Получает доступ к админ-панели
3. Управляет пользователями и постами

## Контрольные вопросы и ответы

### 1. Как обеспечивается безопасность паролей?

Пароли хранятся в виде хеша с использованием password_hash().

---

### 2. Как реализована авторизация?

Используются PHP-сессии ($_SESSION['user']).

---

### 3. Как защищены SQL-запросы?

Используются prepared statements (PDO).

---

### 4. Как реализовано разграничение прав?

Через поле role в таблице users.

---

### 5. Как работает админ-панель?

Доступ предоставляется только пользователям с ролью admin.
